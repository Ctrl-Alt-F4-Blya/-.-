# Premium Design System v7.1 — «Новый дом»

## 1. Итоговая арт-дирекция

Проект переведен в **полностью светлую premium editorial-систему**: теплый ivory-фон, терракотовый акцент, champagne-детали, белые карточки и мягкая сетка. Темные фоны уровня секций убраны с главной, внутренних opening-экранов, FAQ, CTA и footer. Контрастные зоны сохранены только как **внутренние акцентные CTA-карточки**, а не как сплошные темные секции.

Задача версии v7.1 — не переписывать HTML, а стабилизировать дизайн поверх существующих классов и убрать накопленный долг: теперь в CSS один канонический блок `:root`, старые root-переопределения удалены, а цвета в правилах вынесены в переменные.

## 2. Палитра

| Роль | Токен | Цвет | Назначение |
|---|---|---:|---|
| Основной текст | `--ink` | `#0E1821` | Заголовки, основной текст, навигация |

## 3. CSS-архитектура

В `public/assets/css/site.css` оставлен один источник правды:

```css
:root { ... }
```

В него добавлены основные токены и legacy-aliases (`--orange`, `--nd-orange`, `--opening-*`, `--sand`, `--dark-*`), чтобы существующие классы продолжили работать без изменения HTML.

Цвета в правилах приведены к переменным и `rgb(var(--*-rgb) / alpha)`. Старые графитово-серые и оранжевые хардкоды больше не используются в правилах компонентов.

## 4. Переведенные секции

Светлыми стали:

- `.nd-hero`
- `.opening`
- `.faq-classic-hero`
- `.nd-curve--dark`
- `.nd-final`
- `.nd-ownership`
- `.cta`
- `.faq-classic-cta`
- `.site-footer`
- `.section--dark`

Для `.cta`, `.nd-final` и `.faq-classic-cta` сделаны внутренние терракотовые premium-блоки через `.cta-grid`, `.nd-final__grid`, `.faq-classic-cta__inner`. Это дает контраст и выразительность без ощущения темного сайта.

## 5. Типографика

Шкала `--text-*` и модуль 8px сохранены. Локальный FindSans Pro оставлен как основной шрифт, чтобы проект не зависел от внешних CDN.

Заголовки используют легкий вес, плотный letter-spacing и `text-wrap: balance`. Основной текст оставлен с высоким line-height для спокойного чтения инвестиционных материалов.

## 6. Компоненты

### Header

Floating sticky header остается светлым на всех страницах. FAQ-специфичные темные переопределения убраны.

### Hero / Opening

Hero и opening теперь строятся на `--surface / --surface-tint`, текст — `--ink / --ink-soft`, акценты — `--accent`. Метрики и визуальные карточки подняты белыми стеклянными поверхностями.

### Cards

Карточки используют white/glass-поверхности, `--line`, `--shadow-md`, мягкий hover lift и pointer-aware tilt из JS.

### Forms

Inputs, textarea и select работают на светлом фоне, имеют hover/focus/error/success состояния, а keyboard focus вынесен в `--focus`.

### Calculator

Логика калькулятора не изменена. Range-инпуты получают CSS-переменную `--range-progress` из `app.js`.

### FAQ

Мертвые JS-блоки `[data-faq-atelier]` и `[data-faq-wheel]` удалены. Актуальный accordion остался рабочим.

### Footer

Footer переведен на светлый фон с тонкой верхней линией, темным текстом и терракотовыми hover-состояниями.

## 7. SEO и производительность

Добавлено:

- `assets/icons/favicon.svg`;
- `assets/images/og-image.png`;
- favicon во все HTML-страницы;
- canonical, `og:*`, `twitter:*` во все страницы;
- `robots.txt`;
- `sitemap.xml` со всеми 11 страницами;
- `loading="lazy"` и `decoding="async"` на изображения ниже первого экрана.

## 8. Backend

В `server.js` добавлены:

- `helmet()` с безопасной совместимой конфигурацией;
- `express-rate-limit` для `POST /api/leads`: 5 заявок за 10 минут с одного IP;
- `app.set("trust proxy", 1)` для корректной работы за прокси.

Логика Telegram, email, калькулятора и формы заявок не менялась.

## 9. Проверка

Проверено:

- `node --check public/assets/js/app.js`;
- `node --check server.js`;
- CSS parse через `tinycss2`;
- наличие одного `:root` в `site.css`;
- отсутствие старых hardcoded-цветов в правилах компонентов;
- наличие favicon, robots, sitemap, OG/Twitter-тегов.

## v7.1 Light UI Hotfix

Исправления после визуальной проверки:

- Верхняя плашка теперь всегда светлая: ivory/white glass вместо тёмного серого фона.
- Навигация в шапке получила тёмный текст, активное состояние через terracotta underline, без белого текста на светлой поверхности.
- Header CTA стал нормальной terracotta-кнопкой с читаемым белым текстом.

- На мобильных меню остаётся светлым, active/hover получают terracotta-заливку с белым текстом.
- `nd-text-link--light`, outline-кнопки и секции, ранее наследовавшие белый цвет из dark-версий, принудительно переведены в контрастные light-состояния.
- Светлые result-блоки калькулятора получили тёмный основной текст и мягкий вторичный текст.
